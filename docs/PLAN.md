# 기획서 #2 — 한글 데일리 단어 게임 (가칭: 꼬들 / KKODLE)

> 작성일: 2026-07-10 | 상태: Draft v1 | 배포: GitHub Pages 또는 Vercel 단독, 서버·API 비용 0원

---

## 1. 문제 정의

- Wordle은 서버 없는 정적 게임으로 90명 → 2개월 만에 200만 명, 2023년 한 해 48억 플레이를 기록했다.
- 한국어권에는 NYT Wordle 같은 "매일 딱 1판, 결과를 이모지로 공유"하는 습관성 게임의 지배적 사업자가 없다. 기존 한글 워들 클론(예: 꼬맨틀, 한글 워들류)은 유행 후 방치되었거나 UX가 조악하다.
- 심심풀이 수요는 상시 존재하며(출퇴근·점심시간), 공유 기반 게임은 마케팅 비용 없이 성장한다.

## 2. 목표 & 성공 지표

| 지표 | 3개월 | 6개월 |
|------|-------|-------|
| DAU | 500 | 5,000 |
| 결과 공유율 (공유 클릭/게임 완료) | 15% | 25% |
| 재방문율 (7일) | 30% | 45% |
| 광고 수익 | AdSense 승인 | 월 10만 원+ |

핵심 지표는 **공유율**. 공유가 유일한 성장 엔진이므로 공유율이 낮으면 게임 메커니즘을 재설계한다.

## 3. 타겟 사용자

- 1차: 20–30대 모바일 사용자, 카카오톡/인스타 스토리로 결과를 자랑하고 싶은 층
- 2차: 단어 퍼즐 애호가 (십자말풀이, 꼬맨틀 경험자)
- 사용 맥락: 하루 1회, 3–5분, 모바일 세로 화면

## 4. 게임 메커니즘 (핵심 설계)

### 기본 룰 — 자모 분해 워들
- 정답: 두 글자 한국어 명사 (자모 분해 시 5–6 슬롯, 예: "사과" → ㅅㅏㄱㅗㅏ)
- 6번의 시도, 슬롯별 3색 피드백 (정확/포함/없음)
- 하루 1문제, 자정(KST) 갱신, 전 세계 동일 문제 → "오늘 몇 번 만에 풀었어?" 대화 유발

### 차별화 장치 (Wordle 대비)
1. **연속 출석 스트릭** + 통계 (localStorage)
2. **이모지 공유 카드**: `꼬들 #142 4/6 🟩🟨⬛...` + 링크. 카카오톡 공유 시 OG 이미지 자동
3. **어제의 정답 아카이브** 페이지 (SEO 유입용: "꼬들 142 정답" 검색 수요)
4. 시즌 이벤트: 명절·수능 등 시의성 단어

### 확장 모드 (Phase 2)
- 무한 모드 (연습용, 광고 노출 극대화)
- 사자성어 모드 / 초성 퀴즈 모드 — 동일 엔진 재사용

## 5. 기술 스택 & 아키텍처

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Vanilla JS + Vite (또는 Svelte) | 번들 최소화, 정적 출력 |
| 호스팅 | Vercel (무료) | 프리뷰 배포, 커스텀 도메인 |
| 정답 데이터 | 빌드 시 포함된 암호화(XOR/base64) 단어 배열 + 날짜 인덱스 | 서버 불필요. Wordle 원조 방식 |
| 상태 저장 | localStorage | 통계·스트릭·오늘 진행 상황 |
| 단어 사전 (유효성 검사) | 공개 한국어 명사 리스트 (우리말샘 CC 라이선스 확인) 정적 JSON | 클라이언트에서 검증 |
| 공유 | Web Share API + 클립보드 폴백, 카카오 SDK(무료) | 바이럴 핵심 |
| 분석 | Vercel Analytics 무료 티어 또는 GoatCounter(무료) | 비용 0 |

**서버 불필요 원칙**: 정답 판정·통계 전부 클라이언트. 치팅(소스 보기) 가능하지만 Wordle도 동일했고 성장에 무관.

## 6. 페이지 구조

```
/            게임 본체 (오늘의 문제)
/archive     지난 문제 목록 (정답은 D+1 공개) ← SEO 유입
/how-to      게임 방법
/stats       내 통계 (클라이언트 렌더)
```

## 7. 수익화

1. AdSense: 게임 완료 모달 하단 + 아카이브 페이지 (게임 화면 자체엔 광고 금지 — UX 보호)
2. 무한 모드 판 사이 전면 광고 (Phase 2)
3. 커피 후원 버튼 (Buy Me a Coffee / 토스 익명 송금 링크)

## 8. 성장 전략

- 출시일: 커뮤니티 시딩 (에브리타임, 인스티즈, 트위터, 디시 워들 갤러리)
- "꼬들 N 정답" 검색어 선점 — 아카이브 페이지가 자동으로 롱테일 SEO 커버
- 공유 카드에 항상 URL 포함 → 카톡방 유입 루프

## 9. 로드맵

| 주차 | 산출물 |
|------|--------|
| 1주 | 게임 엔진 + 자모 분해 로직 + 단어 리스트 확보 (TDD) |
| 2주 | UI/공유 카드/localStorage 통계, 배포 |
| 3주 | 아카이브·SEO 페이지, OG 이미지, 커뮤니티 시딩 |
| 4주+ | 지표 보고 판단: 공유율 <10%면 메커니즘 피벗, ≥15%면 무한 모드 착수 |

## 10. 리스크 & 대응

| 리스크 | 확률 | 대응 |
|--------|------|------|
| 기존 한글 워들과 차별화 실패 | 중 | 스트릭·아카이브·공유 UX 품질로 승부. 방치된 경쟁작 다수 |
| 유행 실패 (트래픽 0) | 중~상 | 4주 안에 판단, 매몰비용 최소 (총 개발 2주) |
| 단어 리스트 저작권 | 저 | 우리말샘(CC BY-SA) 등 공개 라이선스만 사용, 출처 표기 |
| 자정 갱신 시간대 문제 | 저 | KST 고정 명시 |

## 11. 범위 제외 (Out of Scope)

- 로그인/랭킹 서버, 멀티플레이, 푸시 알림, 앱 출시 — 전부 서버·비용 필요하므로 제외

## 12. 핵심 원칙

1. **서버 제로**: 정답 판정·통계·아카이브 전부 클라이언트. 외부 API 호출 0건.
2. **3초 원칙**: 접속 → 첫 입력까지 3초 이내. 로딩 스피너가 필요한 무게면 실패.
3. **공유가 곧 마케팅**: 결과 공유 UX에 가장 많은 공을 들인다. 공유 텍스트는 카톡에서 깨지지 않아야 한다.
4. **모바일 우선**: 375px 세로 화면 기준 설계, 데스크톱은 확장.
5. **매일 같은 문제**: KST 자정 기준 전 세계 동일 문제. 클라이언트 시계 기준 날짜 인덱스.
6. **점진 개선**: 각 작업 단계는 이전 대비 시각·성능·느낌에서 체감 +12% 이상. 단계마다 브라우저 실검증.

## 13. 제약 조건

| 항목 | 제약 |
|------|------|
| 호스팅 | GitHub Pages 또는 Vercel 단일. 빌드 도구 없이 순수 정적 파일로도 배포 가능해야 함 |
| 비용 | 0원. 유료 API·유료 폰트·유료 에셋 금지 |
| 스택 | Vanilla JS (ES Modules) + HTML + CSS. 프레임워크·번들러 없음 |
| 데이터 | 정답 단어 리스트는 base64 난독화하여 정적 포함. 외부 사전 API 금지 |
| 저장 | localStorage만 사용. 쿠키·서버 저장 금지 |
| 접근성 | 키보드(물리) 입력 지원, 색약 대응 고대비 모드 |
| 브라우저 | 최신 Chrome/Safari/삼성인터넷. IE 미지원 |

## 14. 완성 기준 (Definition of Done)

- [ ] `index.html` 단독 오픈으로 게임 전체 동작 (콘솔 에러 0건)
- [ ] 자모 분해·조합·판정 로직 단위 테스트 전부 통과 (Node로 실행 가능)
- [ ] 6×6 보드: 입력 → 타일 채움 → 제출 → 3색 피드백 → 승/패 판정 동작
- [ ] 가상 키보드 + 물리 키보드 입력 모두 지원, 키보드에 색상 피드백 반영
- [ ] 승/패 시 통계 모달: 플레이 수·승률·연속 기록·분포 그래프 (localStorage 영속)
- [ ] 이모지 결과 공유: 클립보드 복사 + Web Share API, `꼬들 #N X/6` 형식
- [ ] 하루 1문제: 같은 날 재접속 시 진행 상태 복원, 자정(KST) 지나면 새 문제
- [ ] 게임 방법 모달 (첫 방문 시 자동 표시)
- [ ] 375px 모바일 레이아웃 깨짐 없음 + 데스크톱 정상
- [ ] 타일 뒤집힘·키 입력·흔들림(무효 입력) 애니메이션 — "느낌" 기준
- [ ] 다크 테마 기본 + 라이트 토글
- [ ] 페이지 로드 리소스 총합 < 100KB (폰트 제외), 외부 요청 0건
